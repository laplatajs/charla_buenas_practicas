  • Para las clases: ClassNamesLikeThis
  • Para enumerativos: EnumNamesLikeThis
  • Para los metodos: methodNamesLikeThis
  • Para los namespace: foo.namespaceNamesLikeThis.bar

SIEMPRE usar ; en las sentencias. 
JavaScript tiene una funcionalidad poco practica y problemática que es la Inserción
automática de ";" .  Y no funciona como uno espera: 

Si, no usamos ";" , el parseador de JS decide por nosotros donde ubicarlos cambiando
la semantica de ejecucion que creiamos tener.
Expresiones condicionales
NUNCA! usar == o != . Siempre === y !==
  • Objects se evalúan como: true
  • Undefined se evalua como: false
  • Null se evalua como: false
  • Booleans se evalúan como el valor que tengan (true|false)
  • Numeros se evaluan como false si son 0 o NaN; sino devuelven true
  • Strings se evaluan como false si es un string vacio ''; sino devuelven true
Mas información sobre este tema: http://javascriptweblog.wordpress.com/2011/02/07/truth-equality-and-javascript/
undefined y null

JavaScript tiene dos valores distintos para nothing, el más útil de estos dos es undefined.
Mientras que undefined en el contexto del lenguaje JavaScript es muy usado en el sentido 
tradicional como null, el actual null es utilizado en algunos detalles internos de JavaScript 
pero en casi todos los casos, puede ser reemplazado por undefined.

Funciones
Las funciones en JavaScript son las primeras clases de objetos. Esto significa que se
puede pasar como cualquier otro valor. 
La declaración de función anterior se carga así mismo antes de iniciar la ejecución del
programa; por lo tanto, está disponible en todo el scope (ámbito) de la aplicación donde
se ha definido,  aunque hubiera sido llamado antes de definirse en el código.
